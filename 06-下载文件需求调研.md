# 06. 下载文件需求调研

## 调研目的

Phase 1.3 初版只是把赛程数据导出为 CSV、Excel 兼容表和可打印 HTML，满足了“有文件可下载”，但没有充分验证用户真正需要什么。

本次调整目标：

- 先确认用户为什么搜索 World Cup schedule PDF / Excel。
- 再按真实场景设计下载文件。
- 避免为了完成任务而生成低价值文件。

## 真实需求总结

根据 FIFA 官方赛程 PDF、同类赛程下载页面和可文本解析的赛程页面，下载需求主要分为三类。

### 1. PDF 需求

用户搜索 PDF 通常是为了：

- 打印。
- 离线保存。
- 分享给朋友或群组。
- 放入旅行资料夹。
- 快速按日期查看比赛。

因此 PDF 不应该只是网页表格的复制版，而应该：

- 按日期分组。
- 每行显示比赛编号、时间、球队、阶段、城市、球场。
- 控制版面密度，适合打印。
- 包含生成时间和官方来源提醒。
- 不放筛选控件、导航或开发说明。

### 2. Excel 需求

用户搜索 Excel / spreadsheet 通常是为了：

- 筛选球队。
- 筛选城市。
- 按日期排序。
- 做旅行规划。
- 复制到 Google Sheets、Airtable 或自己的数据库。
- 做二次编辑。

因此 Excel 不应该只有一个简单表，而应该包含多工作表：

- README / Source。
- All Matches。
- Group Stage。
- Knockout。
- Venues。
- Groups。

字段需要覆盖：

- Match number。
- Date。
- Day。
- Kickoff ET。
- Kickoff UTC。
- Venue local time。
- Stage。
- Group。
- Team 1。
- Team 2。
- City。
- Host country。
- Stadium。
- Source status。

### 3. CSV 需求

CSV 更适合：

- 数据导入。
- Google Sheets。
- 自动化脚本。
- 后续生成 PDF、Excel、日历文件。

因此 CSV 应该是一张干净的 All Matches 表，不做视觉排版。
