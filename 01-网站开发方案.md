# 01. 网站开发方案

## 1. 网站目标

建设一个围绕 `world cup 2026 schedule` 的英文内容工具站：

```text
World Cup 2026 Schedule Hub
```

网站需要同时满足三类需求：

1. 查询需求：用户要找完整赛程、球队赛程、城市赛程、分组、积分榜、淘汰赛和决赛时间。
2. 工具需求：用户要下载 PDF、Excel，加入日历，按时区查看开球时间。
3. 商业需求：用户要了解门票、直播、旅行、住宿和城市观赛信息。

## 2. 品牌与商标原则

- 品牌名、域名、Logo 不建议直接使用 `FIFA`。
- 页面正文可以自然出现 `FIFA World Cup 2026 schedule` 等搜索短语，但不能造成官方授权误导。
- 门票页必须明确自己是 guide，不是官方售票网站。

建议品牌方向：

```text
World Cup 2026 Planner
2026 Match Schedule Hub
WorldCup26Schedule
```

## 3. 技术架构建议

推荐技术栈：

```text
Next.js
TypeScript
Tailwind CSS
SQLite 或 PostgreSQL
Static Generation + Incremental Static Regeneration
```

推荐理由：

- 赛程、球队、城市、球场、分组等数据适合结构化管理。
- 大部分 SEO 页面可以静态生成，加载速度更好。
- 赛程、积分、直播和门票信息后续需要持续更新，适合增量刷新。

## 7. 页面组件建议

全站通用组件：

- ScheduleTable
- MatchCard
- TeamFilter
- CityFilter
- StageFilter
- TimezoneSelector
- DownloadButtons
- InternalLinkGrid
- FAQBlock
- SourceList
- LastUpdated
- Breadcrumbs

核心交互：

- 用户可以按日期、球队、城市和阶段筛选赛程。
- 用户可以切换本地时区。
- PDF / Excel 下载按钮应清晰可见。
- 每个页面底部提供 FAQ 和相关页面入口。

## 8. 开发阶段

### Phase 1：MVP

目标：上线核心 SEO 页面和基础赛程查询。

交付：

- 首页
- 完整赛程页
- 日期页
- PDF 页
- Excel 页
- 分组页
- 主办城市页
- 门票页
- TV schedule 页
- 基础数据模型
- 基础内链结构

### Phase 2：长尾页面

目标：扩展城市页、球队页和球场页。

交付：

- 重点城市页
- 重点球队页
- 重点球场页
- 城市/球队/球场之间的内链

### Phase 3：工具增强

目标：提升用户留存和外链价值。

交付：

- PDF 动态生成
- Excel 动态导出
- 日历订阅
- 时区转换
- 淘汰赛图
- 积分榜

### Phase 4：新闻与商业转化

目标：承接赛前和赛中热点。

交付：

- 阵容更新
- 球员专题
- 门票更新
- 直播更新
- 今日比赛
- 晋级形势
