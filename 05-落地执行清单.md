
## 8. Google AdSense 技术准备项

目标：
在建站过程中提前预留 AdSense 验证和广告合规所需的技术位置，但不在内容未完成前提交申请。

执行原则：

```text
[ ] 不提前放入虚假的 AdSense publisher ID
[ ] 不在网站内容未完善前提交 AdSense 申请
[ ] 验证代码、ads.txt 和广告代码使用正式账号信息后再上线
[ ] 广告位上线前必须先检查移动端体验和 Core Web Vitals
```

技术准备任务：

```text
[ ] 预留全站 <head> 注入能力，用于 AdSense code snippet 或 meta tag 验证
[ ] 预留 /ads.txt 静态文件路径，正式申请后放入 Google 提供的 ads.txt 内容
[ ] 确认 https://worldcup2026schedule.net/ads.txt 可以被公开访问
[ ] 确认所有核心页面可以输出 AdSense 验证 meta tag
[ ] 准备广告脚本加载开关，避免开发环境和预览环境误加载广告代码
[ ] 为后续广告位预留位置，但首版不影响主要工具、赛程表和下载功能
[x] 添加 Privacy Policy 页面
[x] 添加 About 页面
[x] 添加 Contact 页面
[x] 添加 Disclaimer / Unofficial guide 声明
[ ] 检查页面是否存在低质量、空白、模板化或重复内容
[ ] 检查导航是否能访问所有核心内容页面
[ ] 检查移动端是否无遮挡、无侵入式弹窗、无布局溢出
```

正式申请前检查：

```text
[ ] 首页、Schedule、PDF、Excel、Host Cities、Groups、Dates、TV、Tickets 页面内容完成
[ ] 每个核心页面具备实际工具价值，而不只是文字介绍
[ ] 所有下载文件可访问
[ ] sitemap.xml 和 robots.txt 已上线
[ ] canonical、title、meta description、H1 已完成全站检查
[ ] 官方来源、更新时间、免责声明可见
[ ] 使用正式 AdSense 账号完成站点所有权验证
```
